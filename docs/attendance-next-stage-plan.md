# 考勤下一阶段开发计划

日期：2026-02-02

## 目标
围绕计薪周期、审批联动、异常处理三条主线完善考勤业务能力。

## 阶段 1：计薪周期
- 支持模板自动生成：跨月计薪周期（如 1/26~2/25）
- 支持手工调整：手动补齐/缩短/延长周期
- 输出周期配置说明与示例

## 阶段 2：审批联动
- 将请假/出差/外出/加班审批与日记录对齐
- 冲突策略：出差 + 请假 / 出差 + 加班 / 请假 + 打卡
- 输出审批联动规则模板

### 当前建议（需业务确认）
- 缺卡：自动触发“补卡”提醒，若存在审批单则降级为提示。
- 迟到/早退：若审批单覆盖对应时段，标记为“已处理”，否则提示补充说明。
- 出差/外出：打卡不足 6 小时提示“出差不足阈值”，并允许审批单覆盖。

## 阶段 3：异常处理
- 缺卡、迟到、早退、异常审批提示统一
- 异常日志可追溯（原因 + 规则命中链路）

## 验收策略
- CSV 预览覆盖全量 / 司机 / 保安 / 单休车间
- 规则命中统计 & 异常统计
- UI 验收与 API 验收清单
