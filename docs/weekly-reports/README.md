# V2迁移周报目录

本目录用于存放V2架构迁移的周报。

## 📋 周报规范

### 提交时间
- **截止时间**: 每周五下午5点
- **汇报周期**: 周一至周五

### 命名规范
- 格式: `week-N-YYYY-MM-DD.md`
- 示例: `week-1-2025-10-04.md`, `week-2-2025-10-11.md`

### 填写指南
1. 复制`TEMPLATE.md`模板
2. 重命名为对应周次
3. 按照模板填写各部分内容
4. 提交PR到本目录
5. 通知团队Review

## 🎯 使用建议

### 填写周报前
1. 查看`v2-migration-tracker.md`了解本周完成的任务
2. 收集本周的PR、Issue、测试报告
3. 与团队成员沟通，了解遇到的问题和风险

### 填写周报时
1. 使用数据支撑（PR数量、测试覆盖率、性能指标）
2. 记录关键决策和原因
3. 识别风险并提出缓解措施
4. 突出亮点和成就

### 提交周报后
1. 在Slack `#v2-migration`分享周报
2. 在周五回顾会议上讨论
3. 将重要风险/决策同步到`v2-migration-tracker.md`
