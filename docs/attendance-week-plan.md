# 考勤一周计划（可执行版）

日期：2026-02-02

## 目标
- 形成可复用的考勤规则引擎与模板库
- 打通导入、预览、验收与文档闭环
- 让“司机/保安/单休车间”等组织规则在数据上可验证

## Day 1｜数据与映射梳理
- 复核 CSV 字段映射（考勤组/职位/班次/审批等）
- 从 CSV + userId map 生成司机/保安 userId 列表
- 将 userId 列表写入默认 Rule Set 的 userGroups

## Day 2｜规则模板与组织模板库
- 维护系统模板库（默认/司机/保安/单休车间）
- 维护组织模板库（出差/请假/加班冲突等）
- 模板命中结果与问题清单

## Day 3｜导入与预览能力
- 全量 CSV 预览统计
- 司机/保安专用预览统计
- 记录 rule/warning/status 统计

## Day 4｜计薪周期与配置
- 计薪周期支持：模板自动生成 + 手工调整
- 输出配置说明（计薪周期/班次/节假日）

## Day 5｜UI 验收
- Attendance 页各区块可用性检查
- Admin/Import/Payroll 可视化确认

## Day 6｜API 与集成验证
- 规则模板、规则集、预览 API 验证
- DingTalk 导入参数说明与样例

## Day 7｜文档与验收归档
- 形成开发报告、验收报告
- 记录关键统计指标和已知问题
