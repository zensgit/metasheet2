# DingTalk Staging Evidence Mobile Signoff Gate Design - 2026-04-29

## Goal

Make the final DingTalk staging evidence packet enforce the real mobile
public-form signoff before release handoff.

The existing packet exporter already included P4 remote-smoke evidence and a
validator that rejects incomplete or secret-like artifacts. The mobile signoff
tool was separate, so a final packet could still pass without the nine mobile
public-form scenarios that cover anonymous, login-required, selected-user, and
DingTalk-granted form access.

This slice connects the strict mobile signoff output to the staging packet
export and validation flow.

## Export Contract

The exporter now supports:

```bash
node scripts/ops/export-dingtalk-staging-evidence-packet.mjs \
  --output-dir artifacts/dingtalk-staging-evidence-packet \
  --include-output output/dingtalk-p4-remote-smoke-session/142-session \
  --require-dingtalk-p4-pass \
  --include-mobile-signoff output/dingtalk-public-form-mobile-signoff/142-compiled \
  --require-mobile-signoff-pass
```

`--include-mobile-signoff <dir>` copies one strict mobile signoff directory into
`mobile-signoff/NN-name` inside the packet and records it in
`manifest.json.includedMobileSignoff`.

`--require-mobile-signoff-pass` makes mobile signoff mandatory and rejects the
export if no mobile signoff directory is provided.

## Mobile Gate Rules

When the mobile gate is enabled, every included mobile signoff directory must:

- contain `summary.json` generated by `dingtalk-public-form-mobile-signoff`;
- have `strict=true` and overall `status=pass`;
- have zero validation errors;
- include exactly the nine required mobile public-form checks;
- mark every required check as `pass`;
- include `mobile-signoff.redacted.json`;
- not include raw `mobile-signoff.json`.

The nine required check IDs are:

- `public-anonymous-submit`
- `dingtalk-unbound-rejected`
- `dingtalk-bound-submit`
- `selected-unbound-rejected`
- `selected-bound-submit`
- `selected-unlisted-bound-rejected`
- `granted-bound-without-grant-rejected`
- `granted-bound-with-grant-submit`
- `password-change-bypass-observed`

## Validator Contract

The packet validator now re-checks copied mobile signoff evidence instead of
trusting the exporter manifest.

The validator fails if:

- `manifest.requireMobileSignoffPass=true` but no mobile signoff entry is registered;
- `mobile-signoff/` contains an unregistered directory;
- a registered mobile signoff destination escapes the packet directory;
- copied `summary.json` is missing, malformed, non-strict, or not pass;
- copied `mobile-signoff.redacted.json` is missing;
- copied raw `mobile-signoff.json` is present;
- manifest status and copied summary status disagree;
- any required mobile check is missing or not `pass`.

The validator report now includes `includedMobileSignoffCount` so release
handoff can distinguish packets with no mobile signoff from packets with a
strict mobile signoff bundle.

## README Handoff

Generated packet README files now include:

- an `Included Mobile Public-Form Signoff` section;
- a mobile evidence gate status line;
- recommended operator steps to initialize a mobile kit, generate TODOs, record
  each real mobile result with `--compile-when-ready`, then re-export and
  validate the final packet.

## Secret Handling

This gate preserves the redaction-safe handoff model:

- the raw editable mobile kit is never accepted into the final packet;
- only `mobile-signoff.redacted.json`, `summary.json`, and `summary.md` are copied;
- the existing validator secret scan still runs over the final packet;
- design and verification docs use placeholder paths only.

## Non-goals

- This does not execute DingTalk mobile login or form submission flows.
- This does not create users, grants, forms, records, or group robot messages.
- This does not store real webhook URLs, signing secrets, JWTs, temporary
  passwords, public form tokens, or screenshots.
- This does not replace the human mobile acceptance step; it makes the final
  packet prove that step was completed and compiled strictly.
