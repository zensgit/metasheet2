# T9-W Tier 3 (U-3) un-create ‚Äî development & verification

**Date:** 2026-06-28 ¬∑ **Design-lock:** `multitable-t9w-tier3-uncreate-designlock-20260627.md` (#3325, ratified content-approve; landed after #3323 per gate ‚ë°). **Flag default-off** (`MULTITABLE_ENABLE_CONFIG_UNCREATE`).

## What was built
Config-restore is **revision-scoped**; `classifyRevert` routed every `create`-action revision to `gated` ‚Üí execute `422 RESTORE_NOT_SUPPORTED`. Tier 3 opens the **field/view** subset of that: reverting a config `create` revision now **un-creates** (drops) the entity it created, behind a default-off flag. **Per-revision, not PIT-bulk.** Inverse of Tier 4 (undelete = revert a `delete`).

## Locked decisions ‚Üí implementation
- **U3-L1** ‚Äî `isSupportedUncreate(rev)` (`config-restore.ts`, pure): `action==='create' && entity ‚àà {field,view}`. The route opens this subset; `classifyRevert` stays pure (sheet_config/permission creates still hit the 422).
- **U3-L2** ‚Äî the forward `DELETE /fields/:fieldId` and `DELETE /views/:viewId` cascades extracted into module-level **txn-parameterized** `dropFieldCascade`/`dropViewCascade` (`univer-meta.ts`). The forward routes were refactored to call them with the default `source='mutation'` ‚Äî **behavior-preserving** (the route fetch + `{deleted,sheetId}` response + error handling are unchanged; only the cascade body moved). The un-create execute calls the same helpers. Field cascade = DELETE field + auto-number seq + `meta_links` + strip the column from every `meta_records.data` + order-shift trailing fields + clean every referencing view's filter/sort/group/hidden config ‚Äî **one `batchId`**.
- **U3-L3** ‚Äî un-create records the entity's `delete` revision with `source='restore'` + `restoredFromId = rev.id` (threaded into the entity's OWN revision; cascade revisions stay `'mutation'`). `recordConfigRevision` already supported both columns.
- **U3-L4** ‚Äî opaque **HMAC `planHash`** identity (`restore-preview-identity.ts`, disjoint `type:'config-uncreate-preview'`). `hashUncreatePlan` = `HMAC(secret, canonical({entityAlive, cascadeViewIds, orderShiftIds, columnDataPresent}))`; the raw plan is computed **server-side only** (`computeUncreatePlan`) and is **never** a claim or a response field. HMAC-keyed because the presence bit is ~1-bit (a plain sha256 in the decodable JWT would be brute-forceable). The plan **excludes cosmetic config** (name), so a benign post-create rename does not drift. `computeUncreatePlan` uses the **shared `viewConfigChangedByFieldDrop`** predicate, so the plan's cascade set == the real cascade. Execute recomputes the plan under `FOR UPDATE` and returns **one generic `409 PLAN_DRIFT`** (no sub-reason). `columnDataPresent` is a **coarse presence bit**: plan-drift on column data fires only when it flips **false‚Üítrue** (a column that had no data gets its first value post-preview), NOT on every hidden value change ‚Äî the deliberate no-oracle tradeoff (an exact affected-cell delta would itself leak).
- **U3-L5** ‚Äî preview is **undisclosed**: `{ uncreate:{ entityType, entityId, entityName, note }, previewToken }` ‚Äî names the entity + a destructive-consequence note, **NO counts, NO raw plan fields**. (`canManageFields` doesn't imply unrestricted row read, so any per-cell/row count would leak under read-deny.)
- **U3-L6** ‚Äî flag `MULTITABLE_ENABLE_CONFIG_UNCREATE` (preview + execute ‚Üí 403 when off); entity-type cap **required** (field‚Üí`canManageFields`, view‚Üí`canManageViews`); typed `confirm:'uncreate'` (else 400); the whole drop runs in **one `pool.transaction`** (failure-object pattern ‚Üí zero writes on any guard trip).
- **U3-L7** ‚Äî hard-drop, **no v1 undo**. The `delete` revision lets a *future* Tier 4 recreate the *definition*; column **values are gone** (all post-T by construction). Not described as "recoverable".
- **U3-L8** ‚Äî realtime parity with forward delete: `invalidateFieldCache`/`invalidateViewConfigCache`, no config realtime broadcast.

## Scope
**field:create + view:create un-create only.** OUT ‚Äî each its own design-lock: Tier 4 undelete (revert a `delete`) ¬∑ permission-revert (held entirely) ¬∑ `sheet_config` un-create ¬∑ bulk / PIT config-reset ¬∑ cross-base ¬∑ FE. Enabling `MULTITABLE_ENABLE_CONFIG_UNCREATE` stays a runbook step.
