# 钉钉目录批量处理结果设计

日期：2026-03-26

## 目标

目录管理页已经支持批量开通、批量授权、批量忽略、批量解除绑定、按邮箱批量关联，但管理员执行后只能看到一条全局状态文案，无法快速判断：

- 本次到底处理了多少人
- 哪些成功，哪些失败
- 失败成员是谁，失败原因是什么

本次增强补齐“批量结果摘要与失败明细”。

## 设计

### 1. 保持现有全局状态不变

继续保留顶部 `status/statusTone` 的一行状态提示，不改动现有操作习惯和测试语义。

### 2. 新增最近一次批量结果卡片

位置放在批量工具条下方、成员列表上方，紧贴管理员批量操作区域，避免把结果散落到详情区。

卡片展示：

- 操作名称
- 完成时间
- 总数、成功数、失败数
- 失败成员列表
- 每个失败成员的失败原因
- 失败清单复制
- 失败清单 CSV 导出

### 3. 统一在批量 helper 内产出结构化结果

不为每个按钮单独拼接结果，而是在现有批量处理 helper 中统一记录：

- `mutateSelectedAccounts(...)`
- `batchProvisionUsers(...)`

这样可以覆盖：

- 批量授权钉钉
- 批量取消授权
- 批量忽略
- 批量解除绑定
- 按邮箱批量关联
- 批量开通账号
- 批量开通并授权

### 4. 成功与失败并存时同时保留两层反馈

- 顶部状态条继续给出一句话摘要
- 结果卡片保留结构化失败明细

这样管理员无需翻日志，也不会只看到第一条失败原因。

### 5. 失败清单支持二次处理外带

管理员处理批量失败时，常见动作不是停留在当前页面，而是：

- 复制失败名单发给业务负责人
- 导出失败清单后做二次修正

因此在结果卡片里直接补两个动作：

- `复制失败清单`
- `导出失败 CSV`

前者输出适合聊天/工单的纯文本，后者输出适合留档或后续整理的 CSV。
