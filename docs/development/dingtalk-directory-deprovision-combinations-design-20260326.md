# 钉钉目录离职策略组合化设计

## 背景

现有目录同步只支持单个离职策略值：

- `mark_inactive`
- `disable_dingtalk_auth`
- `disable_local_user`

这会导致管理员无法按业务需要组合动作，例如“目录标记失活 + 禁用钉钉登录”或“禁用钉钉登录 + 停用本地账号”。

## 本次目标

1. 将默认离职策略和成员级覆盖策略从单值改为可组合数组。
2. 保持对线上旧数据的兼容，旧表字段不改名、不强依赖新迁移。
3. 保持 `attendance` 模式下管理员目录入口可用，同时继续拦截非管理员访问。

## 方案

### 1. 策略模型

将策略语义改为动作集合：

- `mark_inactive`
- `disable_dingtalk_auth`
- `disable_local_user`

后端内部使用数组表示，接口返回也使用数组。

### 2. 旧数据兼容

数据库仍沿用现有 `text` 字段：

- `directory_integrations.default_deprovision_policy`
- `directory_accounts.deprovision_policy_override`

兼容规则：

- 旧值 `mark_inactive` 读取时映射为 `["mark_inactive"]`
- 新值写入时统一序列化为 JSON 数组字符串
- `null` 仍表示“成员级覆盖为空，沿用集成默认策略”

这样可以零迁移上线，不阻塞现网。

### 3. 执行语义

目录成员离开同步范围后，目录账号本身仍会被同步逻辑标记为失活。

额外动作按集合执行：

- 包含 `disable_dingtalk_auth` 时，禁用该用户的钉钉登录授权
- 包含 `disable_local_user` 时，停用本地账号，并同时禁用钉钉登录授权
- 空集合表示不执行额外动作

### 4. 前端交互

默认离职策略和成员级覆盖都改为复选框组合。

为了降低管理员手工勾选成本，再补一层常用预设：

- 仅目录失活
- 失活并禁用钉钉
- 全部执行
- 无额外动作

成员级覆盖新增：

- `沿用集成默认策略` 开关

其语义为：

- 开启：提交 `null`，表示不写成员级覆盖
- 关闭：提交当前勾选数组，可为空数组

### 5. 管理员入口回归

继续保留：

- `attendance` 模式下平台管理员可进入 `/admin/directory`
- 非管理员仍被后端 `FORBIDDEN` 和前端特征门控拦截

## 风险与控制

### 风险

1. 旧数据单值与新数组混用导致读写不一致。
2. 成员级 `null` 与空数组语义混淆。
3. CSV 导出和测试夹具仍按字符串处理导致回归失败。

### 控制

1. 后端集中解析旧字符串与 JSON 数组。
2. 前端单独区分“沿用默认策略”和“空动作集合”。
3. 补充组合策略单测、路由测试、页面测试，并执行 OpenAPI 校验。
