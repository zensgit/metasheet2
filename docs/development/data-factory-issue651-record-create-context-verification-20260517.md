## Regression Coverage

### Frontend

`apps/web/tests/multitable-grid.spec.ts`

- `resolveCreateRecordContext()` returns `{ sheetId, viewId }` from `/multitable/:sheetId/:viewId` when workbench refs are blank.
- It does not borrow ids from `/multitable/public-form/:sheetId/:viewId`.
- `grid.createRecord({})` posts a body containing `sheetId`, `viewId`, and `data` when the browser is on a direct multitable route.
- If there is no sheet/view context anywhere, `grid.createRecord({})` records a local error and does not call the API.

### Backend

`packages/core-backend/tests/integration/multitable-record-form.api.test.ts`

- `POST /api/multitable/records` with `{ data: {} }` now returns `400 VALIDATION_ERROR` and the exact message `sheetId or viewId is required`.
- This proves missing context no longer becomes `500 INTERNAL_ERROR / Failed to create meta record`.
